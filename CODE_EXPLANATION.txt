Lines 117-121: def cached_invoke(llm, prompt):
- This is a wrapper function. Everywhere else in the code, instead of doing `llm.invoke()`, we do `cached_invoke()`.

Lines 124-128: _triple_prompt()
- Given text, entity types, and relation types, asks the LLM to extract JSON facts (Head, Relation, Tail, Year).

Lines 130-138: INTENT_PROMPT, KEY_PROMPT, PARAM_PROMPT, RAG_PROMPT

- PARAM_PROMPT: Asks the LLM to answer from its own memory (no documents provided). Used to measure internal confidence.
- RAG_PROMPT: Asks the LLM to answer using only the provided context.

Lines 140-143: EXPLAIN_PROMPT
- [N6 Feature] Instead of just a direct answer, forces the LLM to explain WHY it picked certainty facts and discarded others based on our contradiction rules.

Lines 145-156: SCHEMA_INFER_PROMPT

Lines 429-433: _deg_of()
- Looks up how many connections an entity has in Neo4j. (Used to penalize "hubs" like the word "United States" which connect to everything and aren't useful).

Lines 435-454: _ppr()
- [B4 Feature] Personalized PageRank.
- Starts at the query entities (e.g., "Tesla"). Creates a probability fluid (score) that leaks to connected nodes. It simulates someone clicking random links starting from "Tesla". Nodes closer to "Tesla" get higher scores.

Lines 456-465: _filter_edges()
- [B5 Feature] Compares the query relation ("leads") to the edge relation ("CEO_OF") using strict semantic math (embeddings dot product). Throws away totally unrelated edges (like "BORN_IN").

Lines 467-473: _ref_p()
- [B1 Feature] Relevance probability. Basic check: Does this graph path contain words from the user's question? Higher overlap = higher base score.

Lines 475-479: _temporal_decay()
- [N2 Feature] The older a fact is compared to the current year, the lower its score multiplier becomes uses exponential math (exp(-lambda * age)).

Lines 481-524: retrieve(query)
- The master retrieval function.
- Runs keys, intent, snapshot extraction. Gets edges, filters them, runs PPR.
- Loops through all valid edges. Calculates the massive combined score:
  `combined = (ref_p * hub_pen) * (1 + ppr_avg) * decay * (1 + corroborate)`
- Sorts paths by score, returns top K.

PART 8: MODULE C - CONFLICT RESOLUTION (Lines 526 - 684)
================================================================================

Lines 527-531: _H_str()
- Calculates text entropy. If I ask a question 3 times and get [A, A, A], entropy is 0. If I get [A, B, C], entropy is high. Uses classic Shannon Entropy formula p*log(p).

Lines 533-546: _H_sem()
- Calculates semantic entropy. If I get ["Elon", "Mr Musk", "Jack Dorsey"], standard text entropy says they are 3 unique answers. Semantic entropy converts them to vectors and realizes "Elon" and "Mr Musk" mean the same thing (clusters them), returning a lower, more accurate entropy.

Lines 548-567: _detect_contradictions()
- Scans all retrieved paths.
- If it sees two paths with the exact same relation and tail but different heads (e.g. "Modi -> PM_OF -> India" and "Rahul -> PM_OF -> India").
- It compares their years. The older one is marked `contradicted=True`.

Lines 569-585: EnhancedConflictResolver & _entropy()
- Resolves conflicts. 
- _entropy() asks the LLM a question 3 times (with temperature 0.7) and returns the calculated text/semantic entropy.

Lines 587-595: _confidence()
- [N7 Feature] Calculates a final 0 to 1 confidence score based on:
  1. H_signal (Did the context reduce the LLMs uncertainty?)
  2. Support (Did multiple documents say this?)
  3. Recency (Is the fact new or 10 years old?)

Lines 597-608: _build_chain()
- [N6 Feature] Creates a text string explaining "I selected this fact. I discarded that fact because it was older."

Lines 610-684: resolve()
- The master resolution function.
- 1. Runs contradiction detection to remove stale facts.
- 2. Asks the LLM the question with NO context to see how confused it is natively (H_param).
- 3. Calculates 'tau' (the threshold of how much a document MUST help to be used).
- 4. For each fact, asks the LLM again *with* the fact as context. If the LLMs confusion goes down significantly (delta_H > tau), the fact is deemed corrective and helpful.
- 5. Selects the winning facts.
- 6. Asks for the final answer (using EXPLAIN_PROMPT if contradictions existed).
- 7. Calculates final confidence score and returns all metadata.
