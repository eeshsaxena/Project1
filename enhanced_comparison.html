Scoring Formula Evolution
TruthfulRAG v4 Formula
Combined = Ref(p) × (1 + PPR_avg)
Where: Ref(p) = α×E_cov + β×R_cov · Hub penalty applied
→
Enhanced v5 Formula
Combined = Ref(p) × hub_pen × anomaly_pen × (1 + PPR_avg) × decay × (1 + log(1+support)×w)
Added: decay=e-λ(year_gap) [N2] · log-corroboration [N1] · anomaly penalty [N8]
▲ Key addition: The v4 formula treats a 2014 fact and a 2024 fact identically if Ref(p) and PPR are equal. v5 applies temporal decay so the 2014 fact scores 0.45× the 2024 fact (λ=0.08, gap=10yr).

14 Novel Improvements — Design Rationale
[N1] Corroboration

Cross-Document Support Counting

Every triple extracted from multiple documents increments a support counter stored as a Neo4j edge property. Score: × log(1 + support_count). Gandhi (3 docs) scores higher than Modi (1 doc) even at equal Ref(p).

+evidence signal from document frequency
Module A: _accumulate() → Neo4j: MERGE with support property
[N2] Temporal Decay

Exponential Decay on Age of Facts

The v4 formula ignores the year stored on each edge during scoring. v5 applies decay = e^(−λ × age) where λ=0.08. A 2014 fact scores 0.45× vs a 2024 fact — purely from age, not just contradiction removal.

+temporal sensitivity in base scoring formula
Module B: _temporal_decay() in combined score
[N3] Hybrid Retrieval

BM25 + Semantic RRF Fusion

BM25 misses paraphrased documents (e.g. "head of government" for "Prime Minister"). v5 adds sentence-transformer embedding retrieval and fuses rankings using Reciprocal Rank Fusion: 1/(k+rank_bm25) + 1/(k+rank_sem).

+recall for synonym and paraphrase queries
HybridRetriever class → doc retrieval layer
[N4] Graph Snapshots

Year-Anchored Graph Filtering

For queries anchored to a specific year ("who was CEO in 2019?"), v5 extracts the target year and filters all graph edges to year ≤ target. This prevents future facts from contaminating historical queries.

+historical query accuracy
Module B: _edges(snapshot_year) + _extract_query_year()
[N5] Adversarial Guard

Same-Year Conflict Noise Detection

When [C5] cannot resolve a conflict (two facts share the same year), v5 flags it as potentially adversarial. It then ranks remaining paths by support_count from [N1], using document frequency as the tiebreaker.

+robustness to poisoned or noisy corpora
Module B: _detect_adversarial() → Module C: adversarial sort
[N6] Explanation Chain

Traceable Conflict Resolution Output

When a conflict is resolved, v5 builds a structured explain-chain showing which paths were selected, which were removed, and why (year superseded, score, support). This is injected into the final LLM prompt for an auditable answer.

+transparency and academic verifiability
Module C: _build_chain() → EXPLAIN_PROMPT
[N7] Query Decomposition

Multi-Hop Query Splitting

Queries like "Who founded the company that acquired Twitter?" require multi-hop reasoning. v5 detects trigger phrases and decomposes via LLM into sub-queries, resolving each independently before combining the answer chain.

+multi-hop and chained-reasoning accuracy
Module B: _decompose_query() with DECOMP_PROMPT
[N8] Topology Anomaly

Graph Structure Fake-Fact Detection

Injected fake facts often appear as nodes with very high out-degree but zero in-degree (isolation ratio → 1.0). v5 computes anomaly = out_deg / total_deg and applies a penalty of × (1 − anomaly × 0.5) to suspicious node edges.

+protection against injected misinformation nodes
Module B: _anomaly_score() in combined score calculation
[N9] Query Reformulation

Retry with Rephrased Query on Zero Recall

When BM25+Semantic filtering returns zero relevant edges (hard zero-recall failure), v5 uses the LLM to generate 2 synonym/paraphrase variants of the query and retries retrieval. This turns hard failures into successful second-chance retrievals.

+recovery from zero-recall edge cases
Module B: _reformulate() triggered when fedges=[]
[N10] Negation Filter

Negation-Aware Triple Extraction

Standard KG extraction stores negated facts as positives: "Modi is NOT the PM" becomes (Modi, PM_OF, India). v5 scans the source sentence for negation markers (not, never, isn't, etc.) and discards any triple whose head appears in a negated sentence.

+eliminates false positive triples from negated sentences
Module A: _is_negated() called in build() before _accumulate()
[N11] Entity Salience

Salience-Weighted Ref(p) Scoring

The original Ref(p) gives equal weight to all entities. For "Who is the Prime Minister of India?", "Prime Minister" is highly query-relevant while "India" is background context. v5 computes TF-IDF-style salience weights and uses them to scale each entity’s contribution to Ref(p).

+precision in entity-relation coverage scoring
Module B: _salience() → _ref_p_salience() replaces _ref_p()
[N12] Progressive Sampling

Early-Exit Entropy Sampling

TruthfulRAG v4 always draws n=3 LLM samples per path (fixed cost). v5 starts with n=1: if entropy is clearly high (>0.8 = clear conflict) or clearly low (<0.05 = clear agreement), it exits early. Only ambiguous cases draw the full n=3, saving ~40% of LLM calls on average.

+~40% reduction in LLM sampling calls on typical queries
Module C: _entropy() with progressive n=1→3 logic
[N13] Coherence Check

Post-Build Temporal Coherence Validation

After graph construction, v5 scans for overlapping temporal claims: multiple heads claiming the same (relation, tail, year). e.g., both Gandhi AND Modi are PM_OF India in 2024. These are flagged with a coherence_flag='overlapping' property and deprioritised in retrieval.

+catches impossible simultaneous-year conflicts at build time
Module A: _check_temporal_coherence() called after build
[N14] Confidence Score

Calibrated Answer Confidence (0–100%)

After answering, v5 computes a calibrated confidence score combining: (1) how clearly delta_H exceeded tau (40%), (2) log-scaled corroboration support count (30%), and (3) how recent the winning fact is (30%). Printed alongside every answer for user trust calibration.

+quantified trustworthiness signal per answer
Module C: _confidence() → meta["confidence"] output
