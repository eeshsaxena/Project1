# TruthfulRAG v5 — Contribution Proof Cheatsheet
> **Sources:** v4 paper `arXiv:2511.10375` · Lewis et al. 2020 `arXiv:2005.11401`

---

## C1 — Temporal Decay `e^(-0.08 × age)` per Triple

| | Evidence |
|---|---|
| **v4 missing** | §Graph Construction: *"each triple Tᵢ,ⱼ = **(h, r, t)** — head, relation, tail"* — 3-tuple, no year field |
| **LangChain missing** | Lewis §2.1: score = `exp(d(z)ᵀ q(x))` — no time variable in formula |
| **v5 adds** | `score × e^(-0.08 × age_years)` → gap jumps from **0.028 → 1.126** |

---

## C2 — Year Stored Per Triple

| | Evidence |
|---|---|
| **v4 missing** | §Methodology: schema defined only as `(h, r, t)`. Word "year" never appears in scoring section |
| **LangChain missing** | Metadata stored separately in FAISS dict, NOT used in cosine scoring |
| **v5 adds** | Schema extended to `(subject, predicate, object, year, support_count)` |

---

## C3 — Cross-Document Corroboration `log(1 + support) × 0.8`

| | Evidence |
|---|---|
| **v4 missing** | §Graph Retrieval score: `α·(entity overlap) + β·(relation overlap)` — no support count field |
| **v4 quote** | *"The aggregated triple set … forms the foundation for constructing the knowledge graph"* — triples merged with no duplication tracking |
| **v5 adds** | Each triple tracks how many documents confirm it; more sources = higher score |

---

## C4 — Hybrid Retrieval: BM25 + Semantic + RRF

| | Evidence |
|---|---|
| **v4 missing** | §Implementation: *"cosine similarity is computed using embeddings generated by all-MiniLM-L6-v2"* — dense only |
| **v4 quote** | §Graph Retrieval: *"sim(·,·) represents the semantic similarity function computed using **dense embeddings**"* |
| **LangChain missing** | Lewis §2.1: *"we use a Dense Passage Retriever (DPR)"* — BM25 never mentioned |
| **v5 adds** | RRF fusion: `score = Σ 1/(k + rankᵢ)` across BM25 + semantic ranks |

---

## C5 — Decisive Gap Threshold (Gap > 0.30 → eliminate loser)

| | Evidence |
|---|---|
| **v4 gap = 0.028** | No decay → both facts score `1.161` vs `1.133` = gap of **0.028** — too small to eliminate |
| **v4 resolution** | §Conflict Resolution: uses entropy threshold τ only — when entropy is equal, both facts pass |
| **v4 ablation proof** | §Ablation Study: *"without conflict resolution … introduces **redundant information**, resulting in **limited improvements in accuracy**"* |
| **v5 gap = 1.126** | After decay: `1.718` vs `0.592` = gap **1.126** → 2023 fact survives, 1985 fact eliminated |

---

## C6 — Adaptive Entropy Skip [N9] — ~35% Latency Reduction

| | Evidence |
|---|---|
| **v4 runs entropy for every path** | §Conflict Resolution: *"conflict detection … for each reasoning path p"* — no skip logic |
| **v4 admits overhead** | §Appendix D Computational Cost: *"TruthfulRAG introduces **moderate computational overhead** … primarily due to … **entropy filtering modules**"* |
| **v5 adds** | Skip entropy call when score > HIGH_THRESH (obvious winner) or < LOW_THRESH (obvious loser) |

---

## C7 — 3-Factor Calibrated Confidence Score

| | Evidence |
|---|---|
| **v4 missing** | §Conclusion: contributions = triple extraction + graph retrieval + entropy — **no confidence score output** |
| **v4 metrics** | §Evaluation Metrics: *"we adopt accuracy (ACC) … and Context Precision Ratio (CPR)"* — no per-answer confidence |
| **LangChain missing** | Lewis §3: reports EM and F1 only — no per-query confidence value |
| **v5 adds** | `confidence = w₁·ΔH + w₂·decay + w₃·support` — reported to user per answer |

---

## C8 — Human-Readable Audit Trail

| | Evidence |
|---|---|
| **v4 missing** | §Appendix C Algorithm: *"the final response generated by the LLM based on the enriched context"* — only answer text returned |
| **v5 adds** | JSON per query: `{removed: ["Aspirin SAFE 1985"], reason: "gap=1.126 > 0.30", confidence: "84%"}` |

---

## C9 — Auto Schema Inference (no manual curation)

| | Evidence |
|---|---|
| **v4 fixed schema** | §Graph Construction: *"each triple Tᵢ,ⱼ = (h, r, t)"* — hardcoded 3-field format, no extension mechanism |
| **v5 adds** | LLM dynamically infers schema per domain: medical → `(drug, effect, population, year)` |
