Architecture

How each system processes a query

LangChain RAG retrieves by cosine similarity and passes all chunks to the LLM indiscriminately. TruthfulRAG v5 builds a structured knowledge graph, scores every path by recency and source count, removes contradictions, and adds calibrated uncertainty.

LangChain RAG
Standard Retrieval-Augmented Generation
cosine similarity retrieval, no graph structure
01
Load Documents
Read raw text. No date extraction, no entity recognition.
No temporal metadata
02
Chunk Text
Fixed-size overlapping character windows. Inter-sentence relationships are lost.
Entity context lost at chunk boundaries
03
Embed and Store
Each chunk becomes a dense vector in FAISS or Chroma. Only cosine similarity drives retrieval.
No structural relationships encoded
04
Retrieve Top-K
K nearest vectors are returned. A fact from 1960 and a fact from 2024 rank equally if their embedding is similar to the query.
No conflict detection, no recency weighting
05
Generate Answer
All K chunks passed to the LLM. If two chunks contradict, the LLM makes an unguided choice with no transparency.
No confidence score, no audit trail
TruthfulRAG v5
Conflict-Aware Knowledge-Graph RAG
8 novel features over the v4 baseline
01
Auto Schema Inference [A4+]
LLM reads a sample of documents and infers entity types and relation types. Zero manual configuration across all domains.
Fully domain-agnostic
02
Knowledge Graph Construction [A1-A3]
Extract (head, relation, tail, year) triples. Similar entities merged. Relations normalised. Graph stored in Neo4j with edge weights by source count.
Year and support encoded per edge
03
Hybrid Retrieval [N3]
BM25 and semantic embedding ranks are fused by Reciprocal Rank Fusion. Personalised PageRank then walks the graph structure from seed entities.
Consistently outperforms either signal alone
04
Path Scoring [N1, N2]
Each path scored as: PPR rank x e^(-0.08 * age_years) x log(1 + source_count). Old facts penalised multiplicatively.
Temporal decay is model-agnostic
05
Conflict Detection and Resolution [N6]
Paths sharing the same (relation, tail) but different heads and different years are contradictions. The lower-year path is removed. An explanation chain is built.
Full audit trail of removed facts
06
Entropy Filter [N5]
For each path: sample LLM log-probabilities with and without context. If DeltaH > threshold, the path increases confusion and is discarded.
Removes semantically misleading paths
07
Generate with Confidence Score [N7]
Final answer generated from surviving paths. Confidence = 0.4*entropy_signal + 0.3*source_support + 0.3*recency_factor. Reported with every response.
Honest uncertainty quantification
08
Claim Verification Mode [N8]
Input a declarative statement. System returns SUPPORTED, REFUTED, or UNCERTAIN with graph evidence and a one-sentence reason. Not possible in standard RAG.
Active fact-checking, not just retrieval
09
Adaptive Entropy Sampling [N9]
Paths with score above 0.70 (clearly trustworthy) or below 0.05 (clearly stale) skip the 3x LLM sampling step, saving approximately 30% of inference calls.
~30% latency reduction on CPU
[N8] Claim Verification
SUPPORTED / REFUTED / UNCERTAIN verdicts
Standard RAG can only answer questions by retrieving similar chunks. TruthfulRAG v5 can evaluate a declarative claim against the knowledge graph and return a structured verdict with evidence. This is not available in any standard LangChain pipeline.
# CLI usage
python enhanced_main.py --corpus law.json \
  --verify "The BNS replaced the IPC in 2024"

Verdict : SUPPORTED
Reason : BNS came into force July 2024, repealing IPC
Confidence: 71%
[N9] Adaptive Entropy Sampling
Skip redundant LLM inference on extreme-score paths
Entropy sampling requires 3 LLM calls per candidate path. N9 identifies paths where the score is already extreme (very high or very low) and bypasses sampling entirely. The result is typically 20-30% fewer LLM calls with no measurable change in answer quality.
# Before N9: 5 paths x 3 samples = 15 LLM calls
# After N9: 2 extreme paths skipped = 9 LLM calls

[N9] Adaptive entropy: skipped 2/5 paths
Saved approx. 18 seconds on CPU inference

Applications

Any domain where facts evolve over time

TruthfulRAG v5 requires zero code changes between domains. The schema is inferred from the documents themselves.

Domain: Law
Legal Research
Indian law was overhauled in 2024. Legal databases still contain IPC, CrPC, and the Evidence Act alongside the BNS, BNSS, and BSA.
  • Automatic detection of repealed vs current statutes
  • 164-year temporal gap resolved correctly
  • REPLACED_BY edges identify legislative succession
  • Confidence score warns when context is thin
Domain: Medical
Clinical Decision Support
Medical guidelines change rapidly. A RAG system over mixed-vintage literature can surface outdated protocols as if they were current, with serious consequences.
  • Aspirin / Reye's syndrome safety-critical reversal flagged
  • 83% confidence — highest across all test domains
  • CONTRAINDICATED_FOR edges prevent harmful retrieval
  • Claim verifier can fact-check drug indications
Domain: Politics
Political and Current Affairs QA
News archives span decades. A system asked about the "current" leader must not cite a decade-old fact as the answer.
  • PM of India: 2014 election vs 2024 re-election
  • Twitter CEO: Dorsey vs Musk (2022 acquisition)
  • Temporal snapshot: query anchored to a specific year
  • Support count rewards widely-attested facts
Domain: Science
Scientific Classification QA
Scientific definitions change. Pluto's reclassification is a canonical example of a 76-year-old classification being superseded by a 2006 IAU decision.
  • Pluto: 9 planets (1930) vs 8 planets (2006)
  • Chandrayaan-1 vs Chandrayaan-3: distinct facts retained
  • JWST correctly identified over Hubble as primary
  • 79% confidence — highest in space-domain test
Domain: Journalism
Automated Fact-Checking
A journalist's archive assistant should flag when a claim made in 2025 is contradicted by archived evidence, not silently accept it.
  • [N8] Claim verifier: SUPPORTED / REFUTED / UNCERTAIN
  • One-sentence reason included with every verdict
  • Temporal snapshot: what was stated as fact in a given year
  • Corroboration score rewards multi-source agreement
Domain: Enterprise
Corporate Knowledge Bases
Internal wikis accumulate stale policies, org charts, and product specs. TruthfulRAG can surface current information without re-indexing.
  • Leadership succession via REPLACED_BY relation
  • Revenue figures: older filings correctly demoted
  • Entity merging: Twitter and X Corp unified
  • On-premise deployment: no data leaves the network
