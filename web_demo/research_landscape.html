② Latent Space Probing — ProbeRAG & CLEAR (Nov 2025)

Academic research into detecting conflicts inside the LLM's intermediate layers during generation. Pre-Jan 2026 only.

How It Works
Hidden-State Conflict Detection
During a forward pass through the LLM (e.g., LLaMA-3, Mistral), lightweight probe classifiers are attached to intermediate transformer layers (typically layers 16–24 of a 32-layer model). These probes are trained to output a conflict probability score at each sentence boundary.

③ Production & Community Strategies

Engineering patterns deployed in production RAG systems before January 2026. Primarily documented in r/RAG, LlamaIndex, LangChain GitHub issues, and MADAM-RAG (Oct 2024).

Strategy 1 — Trust Tiers & Freshness Decay
Metadata-Driven Weight Penalisation
Attach a trust score and a timestamp to every chunk at ingestion time. During scoring, multiply the cosine similarity by a freshness weight. High-trust sources (company policy docs) decay slowly; low-trust sources (Slack messages) decay fast.
# Example production freshness formula:
adjusted_score = cosine_score
  × trust_weight # 0.2 (Slack) to 1.0 (Policy Doc)
  × exp(-λ × days_old) # λ hand-tuned per company
Limitation: λ and trust weights are hand-tuned per deployment. No principled way to determine λ=0.001 vs λ=0.08 without domain expertise. Breaks when new document types are added. R
v5 equivalent: v5 uses a principled λ=0.08 (half-life = 8.66 years, calibrated for scientific literature turnover) and infers trust from cross-document corroboration count rather than manual assignment.
Strategy 2 — Diff Summarization (Pre-Generation LLM Step)
LLM-as-Conflict-Detector Before Generation
Before generating the final answer, run a separate LLM call with a prompt like: "Do any of these chunks contradict each other? If yes, list the contradictions." Then use the conflict report to filter or rerank the context.

Limitations:
① +2–4 seconds latency (extra LLM call)
② LLM itself struggles to identify contradictions reliably
③ Does not tell you which document is correct
④ Doubles the LLM cost per query R
v5 equivalent: Entropy filter [N5] serves a similar role — it measures Shannon entropy ΔH to confirm a fact genuinely reduces LLM confusion. But it uses 3 targeted LLM calls per fact (not one megaprompt) and skips them entirely for extreme-score facts [N9], saving ~35% latency.
Strategy 3 — Clarification Routing on Tie
Fail-Safe: Ask the User Rather Than Hallucinate
If two equally-trusted sources contradict each other and the conflict cannot be resolved algorithmically, instead of hallucinating a compromise, the pipeline outputs a clarification question to the user.
# MADAM-RAG-style routing logic (Oct 2024)
if gap < SOFT_THRESH:
    if trust_A == trust_B:
        return clarify_question(query, fact_A, fact_B)
    else:
        return answer(higher_trust_fact)

v5 equivalent: v5's soft threshold (gap 0.10–0.30) flags both facts for entropy measurement rather than routing to clarification. The entropy ΔH measurement determines the winner, so most ties are resolved automatically. Only extreme epistemic uncertainty (truly equal facts, ΔH near 0) would warrant clarification — v5 reports this as low confidence (<30%).
